# 微信 ↔ 本地模型 — 使用指南

**默认 Agent 模式：** 你发微信 → 本地 `qwen2.5:7b` 通过 **PowerShell** 操作电脑 → 结果回到微信。

```
手机微信 ClawBot
    ↕  iLink 官方 API
wechat-local-chat（桥接 + Agent 循环）
    ↕  Ollama tools API
qwen2.5:7b 决定执行什么命令
    ↕
PowerShell（本机完整权限）
```

**安全警告：** Agent 模式可删除文件、改系统设置，仅在你个人可信电脑上使用。命令会记入 `~\.wechat-local-chat\logs\commands.log`。

1. **Ollama 在跑**（你本机已有 `qwen2.5:7b`）
2. **微信 ClawBot 插件已开启**（我 → 设置 → 插件）
3. **cli-in-wechat 已构建**（复用其微信协议层，已完成）

## 在微信里用（Agent 模式）

直接发任务，例如：

```
列出 D 盘根目录有哪些文件夹
在当前目录创建 test-agent.txt，内容是 ok
查看本机 IP 地址
```

默认**结果模式**：先发一条「处理中…」，完成后**只发一行结果**（不显示 PowerShell 命令、不教手动操作）。

示例：`关闭 Chrome` → `处理中…` → `已完成：已关闭 Chrome`

| 命令 | 作用 |
|------|------|
| `/agent` | 终端控制模式（默认） |
| `/chat` | 纯聊天，不执行命令 |
| `/new` 或 `/重置` | 清空对话记忆 |
| `/help` 或 `/帮助` | 显示帮助 |

## 配置

文件：`D:\cursor\61\wechat-local-chat\config.json`

```json
{
  "agentMode": true,
  "model": "qwen2.5:7b",
  "workDir": "D:\\cursor\\61",
  "maxAgentTurns": 8,
  "commandTimeoutSec": 120,
  "maxOutputChars": 4000,
  "notifyEachCommand": true,
  "auditLog": true
}
```

- `agentMode: false` — 启动后默认纯聊天
- 换模型：`ollama pull qwen2.5:14b` 后改 `model`（更大模型 tool calling 更稳）

## 和「转发给 CLI」的区别

| | 本地 Ollama Agent（当前） | CLI 桥接（可选） |
|--|--------------------------|------------------|
| 用途 | 微信指挥本机 PowerShell | 微信指挥 Codex/Claude Code |
| 后端 | Ollama qwen2.5:7b | Codex / Claude Code |
| 启动脚本 | `start-wechat-local-chat.ps1` | `start-cli-in-wechat.ps1` |
| 登录凭证 | `~\.wechat-local-chat\` | `~\.wx-ai-bridge\` |

两套可以同时装，但不要同时运行（都会抢微信消息）。

## 电脑休眠后还能用吗？

已运行一键配置（插电时常开 + 开机自启桥接）：

```powershell
# 普通用户可运行（电源 + 开机自启）
D:\cursor\61\scripts\setup-always-on.ps1

# 建议再以管理员运行一次（关闭网卡节能）
# 右键 PowerShell -> 以管理员身份运行，再执行上面命令
```

配置效果：

| 设置 | 效果 |
|------|------|
| 接通电源 | **永不睡眠**（屏幕 30 分钟可关，电脑继续跑） |
| 合上笔记本盖（插电） | **不睡眠** |
| 用电池 | 60 分钟后睡眠（省电） |
| 登录 Windows | 自动后台启动微信桥接 |
| 日志 | `%USERPROFILE%\.wechat-local-chat\logs\bridge.log` |

**注意：** 电脑必须**开机且联网**。真正关机/休眠后桥接会停，微信暂时收不到回复。要 24/7 使用请**插电、不关机**。

手动后台启动桥接：

```powershell
D:\cursor\61\scripts\start-wechat-local-chat-background.ps1
D:\cursor\61\scripts\status.ps1
```

## 常见问题

**Ollama 未运行**  
另开终端执行 `ollama serve`，或确认 Ollama 应用在后台。

**回复很慢**  
7B 模型在 CPU 上会慢；有显卡会快很多。首条消息可能更慢（加载模型）。

**想重新开始对话**  
微信发 `/new`。
