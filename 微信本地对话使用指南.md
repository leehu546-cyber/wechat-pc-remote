# 微信 ↔ 本地模型对话 — 使用指南

最简单的形式：**你发微信 → 本机 Ollama（qwen2.5:7b）回复 → 回到微信**。

不涉及 Codex / Claude Code，只是纯聊天。

```
手机微信 ClawBot
    ↕  iLink 官方 API
wechat-local-chat（本机桥接）
    ↕  http://127.0.0.1:11434
Ollama · qwen2.5:7b
```

---

## 前置条件

1. **Ollama 在跑**（你本机已有 `qwen2.5:7b`）
2. **微信 ClawBot 插件已开启**（我 → 设置 → 插件）
3. **cli-in-wechat 已构建**（复用其微信协议层，已完成）

---

## 启动（一条命令）

```powershell
D:\cursor\61\scripts\start-wechat-local-chat.ps1
```

首次会出二维码 → 微信 ClawBot 扫码确认。

**不要关这个 PowerShell 窗口。**

---

## 在微信里用

直接打字聊天，例如：

```
你好
今天天气怎么样
用三句话介绍你自己
```

| 命令 | 作用 |
|------|------|
| `/new` 或 `/重置` | 清空对话记忆 |
| `/help` 或 `/帮助` | 显示帮助 |

---

## 配置

文件：`D:\cursor\61\wechat-local-chat\config.json`

```json
{
  "ollamaUrl": "http://127.0.0.1:11434",
  "model": "qwen2.5:7b",
  "systemPrompt": "你是一个简洁友好的中文助手...",
  "maxHistory": 20
}
```

换模型：先 `ollama pull 模型名`，再改 `model` 字段。

---

## 和「转发给 CLI」的区别

| | 本地对话（当前） | CLI 桥接（以后） |
|--|------------------|------------------|
| 用途 | 纯聊天 | 写代码、跑命令、改文件 |
| 后端 | Ollama 本地模型 | Codex / Claude Code |
| 启动脚本 | `start-wechat-local-chat.ps1` | `start-cli-in-wechat.ps1` |
| 登录凭证 | `~\.wechat-local-chat\` | `~\.wx-ai-bridge\` |

两套可以同时装，但不要同时运行（都会抢微信消息）。

## 电脑休眠后还能用吗？

已运行一键配置（插电时常开 + 开机自启桥接）：

```powershell
# 普通用户可运行（电源 + 开机自启）
D:\cursor\61\scripts\setup-always-on.ps1

# 建议再以管理员运行一次（关闭网卡节能）
# 右键 PowerShell -> 以管理员身份运行，再执行上面命令
```

配置效果：

| 设置 | 效果 |
|------|------|
| 接通电源 | **永不睡眠**（屏幕 30 分钟可关，电脑继续跑） |
| 合上笔记本盖（插电） | **不睡眠** |
| 用电池 | 60 分钟后睡眠（省电） |
| 登录 Windows | 自动后台启动微信桥接 |
| 日志 | `%USERPROFILE%\.wechat-local-chat\logs\bridge.log` |

**注意：** 电脑必须**开机且联网**。真正关机/休眠后桥接会停，微信暂时收不到回复。要 24/7 使用请**插电、不关机**。

手动后台启动桥接：

```powershell
D:\cursor\61\scripts\start-wechat-local-chat-background.ps1
D:\cursor\61\scripts\status.ps1
```

## 常见问题

**Ollama 未运行**  
另开终端执行 `ollama serve`，或确认 Ollama 应用在后台。

**回复很慢**  
7B 模型在 CPU 上会慢；有显卡会快很多。首条消息可能更慢（加载模型）。

**想重新开始对话**  
微信发 `/new`。
