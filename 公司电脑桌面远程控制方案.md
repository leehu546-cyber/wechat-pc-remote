# 家里电脑远程控制公司电脑桌面操作方案

目标：回家后，用家里的电脑远程接管公司电脑桌面，能看屏幕、操作鼠标键盘、打开公司电脑上的所有软件。

重要前提：必须符合公司 IT 和安全规定。不要私自绕过公司防火墙，不要把远程桌面端口直接暴露到公网。

## 一、先选方案

### 推荐方案 A：Windows 远程桌面 RDP

适合：

- 公司电脑是 Windows 专业版、企业版或教育版。
- 公司允许远程桌面。
- 你回家后可以连接公司 VPN。

优点：

- Windows 自带。
- 稳定。
- 不需要第三方软件。

缺点：

- Windows 家庭版不能作为被控端。
- 通常必须通过公司 VPN 才能从家里连。
- 远程登录后，公司电脑本机屏幕通常会进入锁定状态。

### 推荐方案 B：RustDesk

GitHub：

- https://github.com/rustdesk/rustdesk
- https://github.com/rustdesk/rustdesk-server

适合：

- 你想像 TeamViewer、AnyDesk 那样远控。
- 公司允许安装远控软件。
- 你希望操作简单，回家直接输入 ID 和密码连接。

优点：

- 开源。
- 支持 Windows、macOS、Linux、Android、iOS。
- 可以直接用官方中继，也可以自建服务器。
- 操作方式最接近“接管桌面”。

缺点：

- 公司安全软件可能拦截。
- 公司可能禁止个人远控软件。
- 用官方中继时，是否允许要看公司政策。

### 推荐方案 C：MeshCentral

GitHub：

- https://github.com/Ylianst/MeshCentral
- https://github.com/Ylianst/MeshAgent

适合：

- 你想自建一个网页远程管理平台。
- 你有一台自己的公网服务器或公司允许的服务器。
- 你想管理多台电脑。

优点：

- 开源。
- 支持远程桌面、文件、终端。
- 被控电脑安装 Agent 后，回家浏览器登录即可管理。

缺点：

- 搭建比 RustDesk 复杂。
- 需要一台稳定服务器。
- 更适合 IT 管理场景。

### 备用方案 D：Apache Guacamole

GitHub：

- https://github.com/apache/guacamole-client
- https://github.com/apache/guacamole-server

适合：

- 你已经有 VPN/RDP/VNC/SSH。
- 想通过浏览器统一访问远程桌面。

优点：

- 浏览器访问。
- 支持 RDP、VNC、SSH。

缺点：

- 搭建复杂。
- 它是网关，不是最简单的一键远控工具。

## 二、最推荐你先试的路线

优先顺序：

1. 如果公司允许 VPN + Windows 远程桌面，优先用 **RDP**。
2. 如果公司允许第三方远控软件，优先用 **RustDesk**。
3. 如果你要长期、多电脑、网页管理，再考虑 **MeshCentral**。

## 三、方案 A：Windows 远程桌面 RDP 操作

### 1. 明天在公司电脑上确认 Windows 版本

公司电脑按：

```text
Win + R
```

输入：

```text
winver
```

或者：

```text
设置 > 系统 > 关于
```

确认版本是否是：

- Windows 10/11 专业版
- Windows 10/11 企业版
- Windows 10/11 教育版

如果是家庭版，不能作为 RDP 被控端。

### 2. 在公司电脑开启远程桌面

路径：

```text
设置 > 系统 > 远程桌面
```

打开：

```text
启用远程桌面
```

记录电脑名称，例如：

```text
DESKTOP-47GBOSK
```

### 3. 允许你的账号远程登录

在远程桌面页面里检查：

```text
远程桌面用户
```

确认你的 Windows 账号在允许列表里。

如果你的账号是管理员，通常默认可以远程登录。

### 4. 设置公司电脑不睡眠

路径：

```text
设置 > 系统 > 电源
```

把接通电源后的睡眠设置为：

```text
从不
```

笔记本要插电，不要合盖，除非已经设置合盖不睡眠。

### 5. 记录公司电脑 IP

公司电脑打开 PowerShell：

```powershell
ipconfig
```

记录 IPv4 地址，例如：

```text
192.168.10.25
```

如果公司有 VPN，回家后通常要先连 VPN，再用这个内网 IP 或电脑名连接。

### 6. 回家后连接

家里电脑按：

```text
Win + R
```

输入：

```text
mstsc
```

在“计算机”里输入：

```text
公司电脑IP
```

或：

```text
DESKTOP-47GBOSK
```

然后输入公司电脑的 Windows 用户名和密码。

如果公司电脑在内网，先连公司 VPN，再打开 `mstsc`。

## 四、方案 B：RustDesk 操作

### 1. 明天在公司电脑安装 RustDesk

下载：

```text
https://rustdesk.com/
```

或从 GitHub Releases：

```text
https://github.com/rustdesk/rustdesk/releases
```

安装 Windows 版本。

### 2. 设置无人值守访问

在公司电脑打开 RustDesk：

1. 找到本机 ID。
2. 设置固定密码，也叫永久密码或无人值守密码。
3. 开启随 Windows 启动。
4. 如果有“安装服务”选项，安装服务。
5. 记录公司电脑的 RustDesk ID。

### 3. 设置公司电脑不睡眠

和 RDP 一样，把接通电源后的睡眠设为“从不”。

### 4. 在家里电脑安装 RustDesk

家里电脑也安装 RustDesk。

打开后：

1. 输入公司电脑的 RustDesk ID。
2. 点击连接。
3. 输入固定密码。
4. 成功后就能看到公司电脑桌面并控制鼠标键盘。

### 5. 如果连接不稳定

可以考虑自建 RustDesk Server。

GitHub：

```text
https://github.com/rustdesk/rustdesk-server
```

但自建服务器需要一台公网服务器或公司允许的中继服务器。普通个人使用，先不要急着自建。

## 五、方案 C：MeshCentral 操作方向

MeshCentral 更适合长期管理多台电脑。

基本流程：

1. 准备一台服务器，安装 MeshCentral。
2. 在 MeshCentral 创建账号。
3. 创建设备组。
4. 下载 Windows MeshAgent。
5. 明天在公司电脑安装 MeshAgent。
6. 回家后登录 MeshCentral 网页。
7. 选择公司电脑，进入 Desktop 页面远控。

GitHub：

```text
https://github.com/Ylianst/MeshCentral
https://github.com/Ylianst/MeshAgent
```

如果你只有一台公司电脑要远控，MeshCentral 有点重，RustDesk 或 RDP 更合适。

## 六、千万不要这样做

不要直接把这些端口暴露到公网：

- RDP：`3389`
- VNC：`5900`
- SSH：`22`

不要在路由器里随便做公网端口转发。

更安全的方式：

- 公司 VPN
- 公司 IT 提供的远程办公入口
- RustDesk 固定密码 + 强密码 + 公司允许
- 自建中继服务器 + 强认证
- MeshCentral + HTTPS + 双因素认证

## 七、明天下班前最终检查

离开公司前一定要测一次：

### 如果用 RDP

- 公司电脑已开启远程桌面。
- 公司电脑不睡眠。
- 你知道电脑名或 IP。
- 你知道 Windows 用户名和密码。
- 家里需要 VPN 的话，确认 VPN 能连。

### 如果用 RustDesk

- 公司电脑 RustDesk 已安装。
- 已设置固定密码。
- 已开启开机启动。
- 已安装服务。
- 记录 RustDesk ID。
- 公司电脑不睡眠。
- 用手机热点或另一台电脑试连一次。

### 如果用 MeshCentral

- MeshAgent 已安装。
- MeshCentral 网页里能看到公司电脑在线。
- Desktop 能打开。
- 公司电脑不睡眠。

## 八、我的建议

如果你只想“回家能操作公司电脑所有软件”，明天优先问公司是否允许远程桌面。

如果允许公司 VPN + RDP，用 RDP。

如果公司允许装远控软件，用 RustDesk，最省事。

如果公司不允许任何个人远控软件，不要硬搞，找 IT 开通公司认可的远程办公方式。
