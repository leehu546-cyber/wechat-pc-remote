  "simple_bypass": true,

### 22 | 优化 | 关闭微信进度文字，只保留最终结果

**日期：** 2026-06-06

**背景：** §21 为修复「做了但不回」启用了 `progress.enabled: true`，导致每条 Agent 任务在微信里出现 3 条消息（`已收到，开始处理` → `处理中：…` → 最终结果）。OpenCode 任务普遍超过 2 秒，`start_delay_sec` 无法抑制进度推送。

**参考：**
- [nanoclaw Discussion #1553](https://github.com/nanocoai/nanoclaw/discussions/1553)：用 iLink `sendtyping` 显示「正在输入」，优于发临时思考文字
- 本仓库 legacy `wechat-local-chat`：`resultOnly: true` + `notifyEachCommand: false`

**改动：**
- `scripts/init-weclaw-opencode.ps1`：`progress.enabled` 改为 `false`；`system_prompt` 删除「show progress promptly」
- `~/.weclaw/config.json` 同步：`progress.enabled: false`
- 保留 `routing.simple_bypass` 与 `SendTypingState`（正在输入）；`cancel_previous` 见 §23–§25

**操作：**
```powershell
D:\cursor\61\scripts\init-weclaw-opencode.ps1
D:\cursor\61\scripts\restart-weclaw.ps1
```

**验证（微信侧）：** 每条任务应只收到 **1 条文字回复**（期间显示正在输入）。可测：`关闭 Claude Codecli`、`天气怎么样？`、`你好`。日志中不应再出现 `已收到，开始处理` / `处理中：`。
