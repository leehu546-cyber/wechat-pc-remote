### 27 | 修复 | OpenCode 空回复（empty response）+ minimal 进度心跳覆盖

**日期：** 2026-06-06

**现象：** 微信收到 `Error: agent returned empty response`；`weclaw.log` 中 `prompt result` 为 `end_turn` 且 `outputTokens>0`，但只有 `tool_call_update`、无 `agent_message_chunk`。

**根因：**
1. OpenCode ACP 竞态：`agent_message_chunk` 常在 `end_turn` **之后**才到达（[opencode#25421](https://github.com/anomalyco/opencode/issues/25421)）
2. 初版仅等 3s 不够（日志 `04:17:41` end_turn → `04:17:44` 仍报错）
3. `progress.mode=minimal` 时 heartbeat 会覆盖「正在调用工具」状态文案

**改动（`weclaw/` 子仓库 `c4b25fe`）：**
- `agent/acp_agent.go`：`end_turn` 后等 **12s** 收迟到 chunk；解析 `tool_call_update` 的 title/rawOutput；仍空则自动 **summary nudge 重试** 一轮
- `messaging/handler.go`：minimal 模式忽略 heartbeat，避免覆盖工具状态

**操作：**
```powershell
cd D:\cursor\61\weclaw && go build -o weclaw.exe .
D:\cursor\61\scripts\restart-weclaw.ps1
```

**验证（2026-06-06 04:26 重启后）：**
